## Literary Necromancy Engine

> "The heads of philologists are stuffed with books to the brim. They see life only through text. And they are proud of it. … Forever gorged and poisoned by literature, they take living life as the continuation of text, as its appendix."
>
> —Vladimir Sorokin

---

### What is this?

`sorokin` is a C/Go literary necromancy engine. It takes your prompts, rips them open like a pathologist with tenure, builds recursive trees of semantic mutations using the geometry of a neural network's latent space, reassembles the corpse through the Dario Equation, and writes a sonnet from the remains.

Zero internet. Zero inference. Zero semantic understanding. Just 79,000 bytes of C, a Go embedding bridge, and the gravitational pull between dead words in a 384-dimensional space.

Named after Vladimir Sorokin, the Russian writer who treats language as flesh to be cut, stitched, and reanimated. `sorokin` does the same thing, but with math.

### The Three-Act Horror Show

#### Act I: The Dissection

`sorokin` takes your prompt and strips it down to the bone:
- Punctuation, numbers, capitalization — gone
- Core words selected by length, rarity, position, and a sprinkle of chaos
- Stopwords rejected. Single letters discarded. What remains are the words that *matter* — or at least the words that think they do.

For each core word, sorokin grows a recursive branching tree of mutations. Width 4, depth 3. Each branch is a semantic neighbor — words that live close to each other in the 384-dimensional embedding space of a 34M parameter neural network trained on human text.

The mutation provider works by **embedding cosine similarity**: given a word, tokenize it, look up its embedding vector, compute cosine distance against all 32,000 vocabulary tokens, return the nearest neighbors. One matrix operation. No forward passes. No inference. Just the geometry of where words landed after training.

When nanollama weights are present, sorokin has access to 32,000 words and their spatial relationships. Without weights, it falls back to 1,158 hardcoded seed words and phonetic neighbors. The autopsy continues either way. The show must go on.

#### Act II: The Reassembly

Leaves collected from all trees — the final mutations at the bottom of every branch — are fed into the **Dario Equation**:

```
p(x|Φ) = softmax((B + α·H + β·F + γ·A + T) / τ)
```

Where:
- **B** = bigram affinity (does this word naturally follow the previous one?)
- **H** = Hebbian trace (has this pair co-occurred before? memory through repetition)
- **F** = Prophecy field (words that were predicted but never spoken — the ghosts)
- **A** = Destiny vector (the gravitational pull of the prompt's original meaning)
- **T** = Trauma (accumulated scars from previous autopsies)
- **τ** = Temperature (how drunk is the surgeon)

The equation samples words one at a time, building a paragraph from corpse parts. The result is grammatically plausible but semantically deranged. Frankenstein's paragraph.

**SuperToken crystallization** (borrowed from [Leo](https://github.com/ariannamethod/leo)): word pairs with high pointwise mutual information fuse into compound units. "dissolution refinement" becomes a single token. The vocabulary grows. The corpse develops its own language.

#### Act III: The Sonnet

The assembled corpse and all 147 leaves are fed into a 14-line Shakespearean sonnet generator:

1. Extract charged words (long + rare) from the corpse
2. Build bigram chains from corpse + leaves + seed vocabulary
3. Match rhyme classes via phonetic fingerprints (last vowel + consonant tail)
4. Assign end-words following ABABCDCDEFEFGG scheme
5. Build each line backward from the rhyme word
6. Shakespearean punctuation: semicolons at quatrain breaks, em-dash before volta

Zero semantic understanding. Just bigrams, phonetic fingerprints, and an unhealthy obsession with structure over meaning.

### Architecture

```
sorokin.c    79K   C autopsy engine, Dario Equation, seed vocab,
                   SuperTokens, bigrams, sonnet, metrics
main.go       8K   CGO bridge, embedding provider, GGUF loader
model.go     18K   Llama 3 forward pass (unused in embed mode)
gguf.go      16K   GGUF v3 parser (F16, Q4_0, Q8_0, Q4_K, Q6_K)
tokenizer.go 10K   BPE tokenizer (SentencePiece + GPT-2)
quant.go     16K   Quantized matmul kernels
quantize.go   5K   F16 → Q4_0 GGUF quantizer
gamma.go      7K   γ personality extraction
engine.go     5K   Inference engine (generative mode)
bridge.c     <1K   CGO function pointer shim
```

The C engine (sorokin.c) is the brain — 1,158 seed words across 11 categories (anatomy, decay, language, philosophy, violence, nature, industry, food, emotion, abstract, culinary), a seed corpus of structural bigrams, Hebbian co-occurrence tracking, prophecy fields, destiny vectors, trauma accumulation, and SuperToken crystallization.

The Go layer loads a nanollama GGUF checkpoint (34M params, Llama 3 architecture) and exposes its 32,000-token embedding table to C via CGO callback. When C needs mutations for a word, Go computes cosine similarity against the full vocabulary in milliseconds. The entire autopsy — trees, corpse, sonnet, metrics — runs in under a second.

### Usage

```bash
# Build with nanollama embeddings (Go + CGO)
make go

# Build standalone C (seed vocab + phonetic neighbors only)
make

# Autopsy
./sorokin "destroy the sentence"

# REPL mode
./sorokin
> bone becomes word and word becomes dust
> the flesh remembers what the mind forgets
> ^D

# Quantize weights (F16 → Q4_0, 66MB → 19MB)
./sorokin -quantize nano-base-f16.gguf nano-base-q4_0.gguf
```

Place `nano-base-q4_0.gguf` (or `nano-base-f16.gguf`) in the working directory or `~/Downloads/nanollama/weights/`. If no weights found, sorokin falls back to seed vocabulary — 1,158 hardcoded words and phonetic neighbor mutations. The autopsy still works. It's just blind instead of seeing in 384 dimensions.

### Resonance Metrics

Every autopsy reports three structural measures:

**Phonetic Diversity** — how many unique sound-patterns exist in the corpse. High diversity means every word has a distinct phonetic fingerprint. Maximum resonance: all words sound completely different. Like a symphony where every note is unique.

**Mutation Depth** — how far the trees mutated from the source. Based on inverse word-length variance. High depth means mutations explored diverse linguistic territory. Low depth means we stayed close to home.

**Bigram Density** — structural echo from the seed corpus. How much of the corpse remembers the morgue. It's not plagiarism. It's ancestral memory.

### The Dario Equation

Named after Dario Amodei. The sampling equation that assembles corpses:

```
p(x|Φ) = softmax((B + α·H + β·F + γ·A + T) / τ)
```

This is not gradient descent. This is not backpropagation. This is gravitational attraction between dead words, weighted by memory (Hebbian), prophecy (what should have been said), destiny (what the prompt wanted), and trauma (what the prompt suffered). Temperature τ controls how far the surgeon's hand shakes.

The equation appears across the [Arianna Method](https://github.com/ariannamethod/ariannamethod.ai) ecosystem — in [Leo](https://github.com/ariannamethod/leo) (literary organism), [DoE](https://github.com/ariannamethod/doe) (democracy of experts), and [dario.c](https://github.com/ariannamethod/dario.c) (the equation's standalone implementation). In sorokin, it reassembles corpses. In Leo, it generates sentences. Same equation. Different organs.
